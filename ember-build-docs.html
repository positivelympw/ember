System Design

Architecture Overview

Ember is a SwiftUI app organized around a state-store pattern. Views observe ObservableObject stores. All intelligence flows through the Claude API.

UserDefaults (on-device persistence)
Core principle: All user context lives on-device in UserDefaults — no central server, no profile database. The only outbound data per API call is: first names, memory summaries, the user's message, and active document text.

CircleStore.swift

The core people graph. Manages CircleMembers and the Drift Engine.

⭕
CircleStore
CircleStore.swift · ObservableObject

Holds the array of CircleMembers. CRUD operations (add, remove, update). Persists to UserDefaults as JSON. Houses the Drift Engine — a function that surfaces who needs attention based on daysSinceContact and nudgeEnabled.

UserDefaults Drift Engine DriftSuggestion Context scoring
The Drift Engine computes daysSinceContact for each member with nudgeEnabled, compares it to that member's DriftLevel.nudgeThreshold, and surfaces a ranked DriftSuggestion list. This powers the DriftNudgeBanner in peopleSurface and the /who command.

Slash command parser and context-aware prompt builder for SMS drafting.

⚡
EmberCommand
Draftengine.swift · enum

Parses typed input for slash commands. EmberCommand.parse() tokenizes the input string and returns a typed command case. ContentView's send handler calls this before every API call to intercept commands.

Static parse() 9 command cases
📝
DraftPromptBuilder
Draftengine.swift · struct

Builds the Claude system prompt for SMS drafting. Combines CircleMember context fields with the DraftIntent to produce a targeted, person-specific message prompt. Output is sent to Claude; response is rendered as a draft bubble.

git clone https://github.com/positivelympw/ember.git
cd ember
open Ember.xcodeproj

# 2. Create Config.xcconfig in Xcode (never commit this)
# Xcode → right-click Ember folder → New File → Configuration Settings File
CLAUDE_API_KEY = sk-ant-your-key-here

# 3. Wire it: Ember project → Info tab → Configurations
#    → Debug → Ember → select Config.xcconfig

# 4. Run: ⌘R (on connected device)
# Dev Pro unlock: tap Ember logo 5 times quickly
